+[ID]
+SCRIPT_NAME: daily_qbittorrent_update_v1.32c_synomap.sh
+PROJECT: SEED_MOVER
+
+[ROLE]
+But métier (en 2–3 phrases max) :
+- Automatise la finalisation quotidienne des torrents achevés dans qBittorrent en les filtrant par catégories approuvées et par ancienneté.
+- Pour chaque entrée valide, crée ou rafraîchit un hardlink vers l'arborescence Synology cible, repositionne la sauvegarde qBittorrent, puis marque le torrent avec le tag SYNO pour suivi.
+
+[CONTEXT]
+- Hypothèse : exécuté via une tâche cron quotidienne sur le serveur hébergeant qBittorrent et l'arborescence Synology montée localement.
+- Lancement typique : script shell autonome (`/scripts/daily_qbittorrent_update_v1.32c_synomap.sh [options]`), souvent appelé sans arguments ou via un wrapper cron/systemd timer.
+- Lit le fichier de configuration `synomap.env` (chemin par défaut défini dans SYNO_ENV) pour charger QB_URL/QB_USER/QB_PASS et autres variables.
+- Consomme le fichier de mapping `MAP_FILE` (par défaut `/data/scripts/mapping_entries.txt`) contenant les colonnes séparées par `|` (identifiant, chemin source, chemin destination).
+- N'écrit rien dans le fichier de mapping ; il crée des répertoires sous `DEST_ROOT` et génère des hardlinks dans `DEST_ROOT/<cat>/<nom_top>`. Interaction avec qBittorrent via API HTTP en utilisant les variables d’environnement QB_URL/QB_USER/QB_PASS.
+
+[INPUTS]
+- Paramètres CLI :
+  - `--allow "cat1 cat2"` : liste d'espaces des catégories qBittorrent autorisées (défaut `sonarr radarr`).
+  - `--min-age SEC` : âge minimum depuis la complétion ou l'ajout (en secondes) avant traitement (défaut 259200 = 3 jours).
+  - `--dest-root PATH` : racine des liens Synology (défaut `/syno/torrents/completed`).
+  - `--limit N` : arrête après N torrents admissibles (0 = illimité).
+  - `--map-file PATH` : fichier de mapping SRC→DST.
+  - `--write` : sans cette option, le script reste en mode dry-run (affiche le plan mais ne crée rien).
+  - `--yes` : indicateur accepté mais non utilisé dans cette version (prévu pour confirmation automatique).
+- Variables d’environnement :
+  - `SYNO_ENV` : chemin d'un fichier `.env` chargé en amont pour définir QB_URL/QB_USER/QB_PASS et éventuellement d'autres paramètres.
+  - `QB_URL`, `QB_USER`, `QB_PASS` : credentials/API endpoint de qBittorrent (obligatoires, sinon arrêt immédiat).
+  - `ALLOW`, `MIN_AGE`, `DEST_ROOT`, `LIMIT`, `MAP_FILE` peuvent aussi être définies dans l'environnement pour surcharger les valeurs par défaut.
+- Contraintes d'entrée :
+  - `mapping_entries.txt` doit être un fichier texte avec séparateur `|` où la deuxième colonne correspond au chemin source absolu (tel que qBittorrent rapporte) et la troisième au chemin destination final.
+  - Le montage Synology et les chemins `DEST_ROOT`/`DST` doivent résider sur le même device pour autoriser les hardlinks.
+
+[OUTPUTS]
+- Fichiers : création des répertoires `DEST_ROOT/<cat>` (si `--write`) et des hardlinks individuels `DEST_ROOT/<cat>/<top_file>` pointant vers le chemin `DST` issu du mapping.
+- Actions qBittorrent :
+  - Déplacement de l'emplacement de sauvegarde (`torrents/setLocation`) vers `DEST_ROOT/<cat>`.
+  - Ajout du tag `SYNO` au torrent traité.
+  - Lancement d'un recheck si un nouveau lien a été posé.
+- Journaux : sortie standard détaillant les torrents inspectés, les raisons de skip (`SKIP:category`, `SKIP:age`, etc.) et les opérations de lien.
+
+[FAILURE_MODES]
+- Variables qBittorrent manquantes (QB_URL/QB_USER/QB_PASS) → arrêt immédiat avec message `[ERROR] ... not set` et exit code ≠ 0.
+- Fichier de mapping absent ou introuvable → arrêt (`MAP_FILE not found`).
+- Échec d’authentification qBittorrent → message `qB login failed` et terminaison.
+- Absence de timestamp valide (ni `completion_on` ni `added_on`) → torrent ignoré (`SKIP:no_time`).
+- Torrents trop récents → ignorés (`SKIP:age`).
+- Catégorie non autorisée → ignorée (`SKIP:category`).
+- Fichier TOP introuvable ou mapping manquant → lignes `[MAPMISS]` et aucun lien créé.
+- Volumes source/destination sur devices différents → `[CROSS-DEV]` (impossibilité de hardlink) et torrent sauté.
+- Échec `stat` sur chemins destinés → `[NOSTAT]` et skip.
+- Si les opérations qBittorrent (setLocation, addTags, recheck) échouent, l'erreur n'est pas contrôlée mais se reflète dans les logs `curl` si exécuté en mode verbeux ; le script continue généralement.
